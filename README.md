# Mochi-phone

手机上传前后端版。

## 本版功能

此版本包含：

- 第三阶段剧情运行器：剧情正文、自定义行动、推进剧情、回合/地点/时间/体力、当前事件

- 第二阶段手机桌面：联系、朋友圈、日记本、备忘录、地图、应用市场

- 角色聊天
- 小手机第一阶段沉浸系统
- 角色关系值、心情值、关系阶段
- 角色主动消息
- “我们的回忆时间线”
- 聊天页自适应布局，最后一条消息不会被输入框遮住
- 豆子扣费与充值模拟
- 空回复 / 上游异常自动退豆
- 记忆系统
- 删除对话时自动清理对应自动记忆
- 导出数据备份
- 导入数据恢复

## 记忆系统

记忆页支持：

- 手动新增记忆
- 删除单条记忆
- 按角色保存记忆
- 按类型保存记忆：用户资料、角色关系、事件、偏好、禁忌、剧情
- 对话中自动沉淀重要记忆
- 每次聊天自动把相关长期记忆注入角色 Prompt

自动记忆会记录来源对话和来源消息。

删除单条消息、删除角色对话、删除角色时，会同步清理由这些对话自动生成的记忆。手动新增的记忆不会被自动删除，避免误删用户主动保存的内容。

## 小手机第二阶段

聊天页已删除上方状态栏，角色选择改回适中尺寸，并固定在消息列表上方，避免被聊天气泡遮挡。

底部保留“手机”入口，手机页参考 EVEChat 做成简洁桌面，不做 JRSY 那种复杂全功能系统，也不重复已有的聊天和记忆模块：

- 论坛：改为“回响环廊”结构，包含潮汐广场、深水区、暗房、今日浪潮、回响晚报，以及共鸣/留声/扩音互动数据
- 世界书：保存世界观、地点、组织、重要设定的预留入口
- 纪念日：展示和当前角色认识天数，并预留重要日子

已删除手机桌面里的重复入口：Chat、短信、设置、记忆。

记忆页保留“我们的回忆时间线”，会同时展示：

- 主动消息事件
- 聊天后产生的关系事件
- 自动沉淀的长期记忆

## 当前调整

已删除底部“剧情”入口和剧情模块。当前阶段先把小手机做稳，避免一次做太多导致结构混乱。

## 稳定性

此版本保留空回复退豆逻辑：

- 上游流式内容被网络分块切开时，会用缓冲区继续解析，减少空回。
- 如果上游整次没有返回有效文本，会自动退回本次扣除的豆子。
- 如果上游请求失败，也会自动退回本次扣除的豆子。
- 退豆会写入消费明细。

## 数据备份

此版本新增自动保存和手动备份：

- 豆子余额、消费记录、记忆会自动保存到服务端 `.data/mochi-phone-data.json`
- 记忆也会同步保存一份到手机浏览器本地，服务端重启或重部署后可作为兜底显示
- 服务重启后会自动读取该文件，避免刷新后回到 30 豆
- `DATA_DIR` 可自定义数据目录，默认是 `.data`

- 导出数据备份
- 导入数据恢复

用户换浏览器、换手机前，可以在“我的”页面点击“导出数据备份”，保存 JSON 文件。

换设备后打开网站，在“我的”页面点击“导入数据恢复”，选择备份 JSON 文件，即可恢复：

- 用户 ID
- 角色
- 对话
- 装扮
- 豆子余额
- 消费记录
- 记忆数据

Render：

- Build Command: `npm install`
- Start Command: `npm start`

环境变量最重要只需要：

```env
UPSTREAM_API_KEY=你的真实上游API密钥
```

此版本已内置：

```env
UPSTREAM_API_BASE=https://az.zlapi.vip/v1
UPSTREAM_MODEL=[0.005]自营伪流/gemini-2.5-flash
```
